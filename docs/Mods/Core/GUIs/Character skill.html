Character Skill Editor

The Character Skill Editor is used to create skills with multiple levels of evolution. Skills can affect the user as well of an enemy.

Levels:

Each skill can have as many levels as needed but only one is active at a time meaning that each level is unaffected by the other levels.

Each level contains a list of Activations.

Selecting a different Level will update the Activations list to match which Activations the Level contains

Activations:

Activations are simply a container for a list of Requirements and Effects.

When every requirements of an Activation are met, it will activate its effects. This mean a skill can have multiple Activation and have only one of them active, this way it's possible to have one Activation with only a passive requirement to have an effect active at all time and and have an other Activation that activate under a different requirement. All Activations can be active at the same time.

Requirements:

Requirements are a list of conditions that all need to be met before activating the Effects defined by the selected Activation.

There is no limit to the amount of Requirements that can be used. Some Requirements are passive and will be considered met as long as the condition is true. If used alone those Requirements will launch the Effects for every actions the player do and may cause unexpected behavior if the Lifetime is not set to Permanent.

Some Requirements will also need extra parameters to work and the bottom box will update to show what can be changed.

Effects:

List of effects that will be activated by the selected Activation.

New Effects can be added to the list with the Add button.

Effects can be removed from the list with the Remove Button.

The selected Effect can be changed with the dropdown list.

Each Effect can be applied to the user and/or an enemy. They can be used to increase or decrease statistics or give new abilities.

Some Effects will also need extra parameters to work and the bottom box will update to show what can be changed.

Affected types:

Defines who will be affected by the Effects. The affected types are context sensitive if used on an enemy.

There can be more than one affected types selected.

If activated from a battle, it will automatically use the Effects on the enemy.

  • Self: Used on the leader of the current Squad.
  • Squad: Used on the leader and wingmans of the current Squad.
  • Enemy: Used on the leader of the targeted enemy.
  • Enemy Squad: Used on the leader and wingmans of the targeted enemy.
  • ALL: Used on every Squad on the map.
  • ALL allies: Used on every Squad of the same team on the map.
  • ALL enemies: Used on every Squad of a different team on the map.
  • Aura: Used on every Squad of the same team within a distance from the user.
  • Enemy aura: Used on every Squad of a different team within a distance from the user.
  • Radius: Only used when a type of Aura is selected. Defines the radius used by the Aura to detects targets.

Lifetime types:

Defines how long the Effects will last. If needed, the Effects can be deactivated when a Requirement fails. This way you can have a Permanent Effect stop working after a Requirement is met.

  • Permanent: Once activated, the Effect will never expire.
  • Turns: Once activated, the Effect will expire after a number of turns.
  • Battle: Once activated, the Effect will expire after a number of battles.
  • Stacking: The Effect can be activated a number of times to multiply its effect.
  • Reset when Requirements fail: Will deactivate the Effects if one of the Requirements fail.
