Deathmatch Skill Requirements

Deathmatch Skill Requirements are Requirements that will only work if used with a Deathmatch Map.

Some Requirements have a Before and After version, the Before means that it will activate before damage computation, while after means after the damage computation.

 

Battle Start Requirement:

Fields:

  • None

Description:

This Requirement is activated for every Units of every Squads before any damage of accuracy calculations are made. This inlude support Squads and defending Units.

This Requirement can not be used in combination of the VS Requirements since the Units are not attacking at this point.

Attack Requirement:

Fields:

  • None

Description:

This Requirement is activated for each Unit that is attacking. The order of activation follow the combat logic, as such, counter attacking Unit will have this effect activated after the attacking Unit. Support attack Unit will activate after counter attack as well.

This Requirement can be used in combination of the VS Requirements since a Unit is being targeted.

Getting Attacked Requirement:

Fields:

  • None

Description:

This Requirement is activated for each Unit that is getting attacked. The order of activation follow the combat logic, as such, the Unit being targeted by the counter attack will have this effect activated after the attacking Unit. If a support defend Squad is used, it will activate this effect instead of the original target.

This Requirement can be used in combination of the VS Requirements since a Unit is being targeted.

VS Pilot Requirement:

Fields:

  • Pilot Name: The name of the enemy Pilot

Description:

This Requirement is activated when a Unit is fighting against a Pilot with the corresponding name.

This Requirement is passive and must be used with a combinason of a manual Requirement to avoid the skill being activated multiple times in the same battle.

VS Unit Requirement:

Fields:

  • Unit Name: The name of the enemy Unit

Description:

This Requirement is activated when a Unit is fighting against a Unit with the corresponding name.

This Requirement is passive and must be used with a combinason of a manual Requirement to avoid the skill being activated multiple times in the same battle.

Attack Used Requirement:

Fields:

  • Attack Name: The name of the Attack

Description:

This Requirement is activated when a Unit if fighting with an Attack with the corresponding name.

This Requirement is passive and must be used with a combinason of a manual Requirement to avoid the skill being activated multiple times in the same battle.

Attack Defended Requirement:

Fields:

  • Attack Name: The name of the enemy Attack

Description:

This Requirement is passive and must be used with a combinason of a manual Requirement to avoid the skill being activated multiple times in the same battle.
