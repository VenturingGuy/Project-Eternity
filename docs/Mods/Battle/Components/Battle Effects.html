Battle Effects

Battle Effects are effect that can be used on any map based on the Battle Map such as the Deathmatch Map.

Status Effect:

Fields:

  • Status type: Which status to affect, can be MEL, RNG, DEF, SKL, EVA or HIT.
  • Value: How much to increase or decrease the status. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect adds a temporary bonus value to the stats of a Character.

Unit Stat Effect:

Fields:

  • Number type: Absolute for a fixed number, Relative for a percent of the maximum value of a stat. (1 being 100%, 0.50 being 50%)
  • Unit stat: Which status to affect, can be MaxHP, MaxEN, Armor, Mobility or MaxMV
  • Value: How much to increase or decrease the status. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect adds a temporary bonus value to the stats of a Unit.

Will Effect:

Fields:

  • Number type: Absolute for a fixed number, Relative for a percent or the current Will. (1 being 100%, 0.50 being 50%)
  • Value: How much to increase or decrease the Will. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the Will of a Character.

Will Limit Effect:

Fields:

  • Value: How much to increase or decrease the Will limit. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the maximum Will of a Character.

Fields:

Description:

This Effect changes the SP of a Character.

Max SP Effect:

Fields:

  • Value: How much to increase or decrease the max SP. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the maximum SP of a Character.

Hit Rate Effect:

Fields:

  • Value: Fixed value used to increase or decrease the accuracy of an Attack. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the Attack's HIT value with a fixed number when attacking, is part of the "Other" value in the Accuracy formula.

1 is equivalent to a 1% increase.

Critical Hit Rate Effect:

Fields:

  • Value: Fixed value used to increase or decrease the chances of a critical hit. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the chances of getting a critical hit.

1 is equivalent to a 1% increase.

Critical Hit Always Succeed Effect:

Fields:

  • None

Description:

This Effect guarantee that a critical hit will happen.

Critical Hit Always Fail Effect:

Fields:

  • None

Description:

This Effect guarantee that a critical hit will not happen. This Effect takes priority over any other critical effect.

Evasion Effect:

Fields:

  • Value: Fixed value used to increase or decrease the evasion when attacked. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the final evasion value by a fixed number, is part of the "Other" value in the Evasion formula.

1 is equivalent to a 1% increase.

Base Damage Effect:

Fields:

  • Number type: Absolute for a fixed number, Relative for a percent. (1 being 100%, 0.50 being 50%)
  • Value: How much to increase or decrease the base damage. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the base damage value by a fixed number. The base damage is the damage without the critical bonus if there is one.

Absolute damages are not affected by other damage modifiers.

Relative damage is affected the the Damage Taken Effect.

Damage Taken Effect:

Fields:

  • Number type: Absolute for a fixed number, Relative for a percent. (1 being 100%, 0.50 being 50%)
  • Value: How much to increase or decrease the base damage taken. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the base damage taken.

Absolute damage will take priority over every other damage Effects. Absolute damage decide how much damage the defender will take and ignores criticals.

Relative damage percent is multiplied by the attacker's Base Damage relative effect.

The base damage is the damage without the critical bonus if there is one.

Final Damage Effect:

Fields:

  • Number type: Absolute for a fixed number, Relative for a percent. (1 being 100%, 0.50 being 50%)
  • Value: How much to increase or decrease the final damage. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes the final damage value by a fixed number. The final damage is the damage after the critical bonus if there is one.

Absolute final damages are affected by relative final damage Effects.

EN Cost Effect:

Fields:

  • Value: Fixed value used to increase or decrease the EN cost of an attack. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes EN cost of an attack that uses EN. It is possible to make attacks give EN instead of using it if the EN Cost Effect is too high.

Max Ammo Effect:

Fields:

  • Value: Fixed value used to increase or decrease the max ammo of all attacks. Can be a positive or negative number. Is compatible with the formula parser.

Description:

This Effect changes max ammo for all attack, when used it will also increase the ammo count of each attack based on the value used.

Unit Terrain Effect:

Fields:

  • Terrain: Name of the terrain attribute to affect.
  • Number type: Absolute to set the attribute for a fixed value, Relative for an increase.
  • Value: How much to increase or decrease the attribute of the terrain. Can be a positive or negative number. Is compatible with the formula parser.
  • Can downgrade: Only used if Absolute number type is selected, prevent the attribute for being downgrade if the value is lower than the current Unit's attribute.

Description:

This Effect changes the terrain's attribute for a Unit.

Unit Movement Effect:

Fields:

  • Use Air: Allow or not a Unit to move through air.
  • Use Land: Allow or not a Unit to move through land.
  • Use Sea: Allow or not a Unit to move through sea.
  • Use Space: Allow or not a Unit to move through space.

Description:

This Effect changes where a Unit can go.
